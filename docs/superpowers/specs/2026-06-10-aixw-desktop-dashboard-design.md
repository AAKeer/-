# AIXW 桌面实时看板设计文档

## 目标

构建一个 Windows 桌面软件，用于读取当前已登录 AIXW 账号的数据，并每 15 秒刷新一次，在桌面窗口中展示核心使用指标。

这个项目的目标是“做出一个好用的工具”，不是为了贴合当前 Java 学习路线。因此技术选型优先考虑成熟、通用、桌面交付体验好、界面效果容易做出来。

## 技术方案

第一版采用：

- 桌面壳：Electron
- 界面：React + TypeScript
- 构建工具：Vite
- 打包工具：electron-builder
- 样式：普通 CSS 或 CSS Modules

选择 Electron 的原因：

- Windows 桌面应用资料多，打包方案成熟。
- 适合做深色卡片式仪表盘界面。
- 可以内置登录窗口，保存 AIXW 登录态。
- 主进程可以统一负责请求接口和管理登录态，界面层只展示数据。

## 展示数据

桌面看板第一版展示当前账号的这些指标：

- 余额
- API 密钥总数
- 启用中的 API 密钥数量
- 今日请求数
- 今日消费
- 今日 Token，单位换算成“万”
- 累计 Token，单位换算成“万”
- 缓存命中率
- 最近刷新时间
- 加载状态和错误状态

第一版刷新间隔固定为 15 秒。

## AIXW 接口

通过分析 AIXW 当前前端代码，第一版会使用这些登录后接口：

- `GET /user/profile`
  - 读取当前用户信息和余额。
- `GET /keys?page=1&page_size=100`
  - 读取 API 密钥列表，用于统计密钥总数和启用数量。
- `GET /usage/dashboard/stats`
  - 读取今日请求、今日消费、今日 Token、累计 Token 等仪表盘统计数据。
- `GET /usage/stats?start_date=YYYY-MM-DD&end_date=YYYY-MM-DD`
  - 如果仪表盘统计接口没有缓存字段，则用这个接口补充缓存相关数据。

所有接口都需要登录态。软件优先读取 JSON 接口，不通过截图识别或网页文字抓取来读取数据。

## 登录方式

第一版使用内置登录窗口：

1. 用户点击“登录”。
2. Electron 打开 AIXW 登录页：`https://aixw.org/login`。
3. 用户直接在 AIXW 页面完成登录。
4. Electron 保存登录后的会话 Cookie。
5. 后续刷新数据时，软件使用这个登录态调用接口。

软件不会要求用户把密码、API Key 或原始 Cookie 写进代码里。

退出登录时，软件清除 `aixw.org` 相关登录态。

## 数据流

1. 界面启动后，请求主进程读取看板数据。
2. 主进程携带登录态调用 AIXW 接口。
3. 主进程把多个接口结果整理成统一的看板数据对象。
4. React 界面只接收整理后的数据并展示。
5. 软件每 15 秒自动刷新一次。
6. 用户点击手动刷新时，立即重新读取，并重置刷新倒计时。

主进程负责网络请求和登录态管理。渲染进程不直接保存 Cookie、密码或账号凭证。

## 缓存命中率

如果接口直接返回缓存命中率字段，则优先使用接口字段。

如果没有直接字段，则使用下面的方式计算：

```text
缓存命中率 =
  total_cache_read_tokens /
  (total_input_tokens + total_cache_creation_tokens + total_cache_read_tokens)
  * 100
```

如果分母为 0，则缓存命中率显示为 `-`。

后续可以在鼠标悬浮提示里显示缓存读取 Token、缓存创建 Token 和参与计算的总输入 Token。第一版可以先只展示百分比。

## 数值格式

- 余额：`$47.25`
- 消费：`$4.0157`，默认保留 4 位小数
- 请求数：`1,227`
- Token 换算成“万”：
  - `41000` 显示为 `4.10 万`
  - `440000` 显示为 `44.00 万`
- 缓存命中率：`35.7%`
- 未知或缺失数据：`-`

## 界面设计

第一屏直接进入看板，不做宣传页或介绍页。

整体风格：

- 深色背景
- 卡片式指标布局
- 每张卡片包含图标、标题、主数值和辅助说明
- 顶部显示应用名称、登录状态、刷新按钮、最近刷新时间
- 刷新失败时显示错误提示，但保留上一次成功读取的数据

第一版卡片：

- 余额
- API 密钥
- 今日请求
- 今日消费
- 今日 Token
- 累计 Token
- 缓存命中率
- 平均响应或性能指标，如果接口中可用

## 异常处理

- 未登录：显示登录入口，暂停自动刷新。
- 登录过期：提示“登录已过期”，引导重新登录。
- 网络失败：显示错误提示，保留上一次成功数据。
- 接口字段变化：能展示的字段继续展示，缺失字段显示 `-`。
- `401` / `403`：按登录失效处理。
- `429` 或服务端错误：显示本次刷新失败，等待下一次 15 秒刷新重试。

## 验证标准

第一版完成时需要验证：

- 软件可以在 Windows 上启动。
- 可以打开内置登录窗口并登录 AIXW。
- 登录后可以读取当前账号数据。
- 数据每 15 秒自动刷新。
- 手动刷新可用。
- 退出登录会清除登录态。
- 缓存字段缺失时软件不会崩溃。
- 缓存命中率在分母为 0 时显示 `-`。
- 桌面常见宽度下，文字不溢出、不重叠。

## 第一版不做的内容

- 公开通道状态监控。
- 创建、删除、修改 API 密钥。
- 展示完整用量表格。
- 导出 CSV。
- 托盘常驻模式。
- 自定义刷新间隔。
- 多账号切换。
- 云端同步。
